# UX Decisions

## Decision Log Status

This document will track product-design decisions as the project evolves. Initial entries are directional and should be updated when wireframes and UI are created.

## Decision 1: Start With Documentation Before UI

Rationale:

- This is a product design portfolio project, so problem framing and design reasoning should come before interface polish.
- Documentation helps future UI choices connect to user needs and UX laws.

Impacted principles:

- Tesler's Law.
- Progressive Disclosure.
- Aesthetic-Usability Effect.

## Decision 2: Use a Renewal Queue as the Core Workflow

Rationale:

- Renewal work is deadline-driven and action-oriented.
- A prioritized queue reduces the need for users to search across all subscriptions first.

Impacted principles:

- Hick's Law.
- Serial Position Effect.
- Recognition over Recall.

## Decision 3: Make Risk and Savings Visually Distinct

Rationale:

- High-risk renewals and savings opportunities are exceptions that need attention.
- These signals should stand out without overwhelming the whole interface.

Impacted principles:

- Von Restorff Effect.
- Law of Common Region.
- Aesthetic-Usability Effect.

## Decision 4: Keep Pending Approvals Visible

Rationale:

- Incomplete renewal decisions can stall when ownership is unclear.
- Persistent approval visibility supports follow-through.

Impacted principles:

- Zeigarnik Effect.
- Recognition over Recall.
- Law of Proximity.

## Decision 5: Use Confirmation for High-Impact Actions

Rationale:

- Renewing, canceling, or escalating a subscription can affect budget and team operations.
- Confirmation summaries help prevent accidental decisions.

Impacted principles:

- Error Prevention.
- Peak-End Rule.
- Fitts's Law.

## Open Decisions

- Should opportunity review be a standalone workflow or part of the renewal detail page?
- Which mock data fields are necessary for a credible portfolio demonstration?
- How should incomplete usage signals be represented without implying real integrations?
