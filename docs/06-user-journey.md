# User Journey

## Scenario

A finance operations manager starts the week by reviewing upcoming SaaS renewals and deciding which subscriptions need action.

## Journey Stages

### 1. Orient

The user opens Renewly and scans dashboard summaries for upcoming renewals, high-risk items, pending approvals, and savings opportunities.

Design focus:

- Use Serial Position Effect to place priority signals early.
- Use Von Restorff Effect to distinguish urgent and high-value items.
- Use Progressive Disclosure so the dashboard stays scannable.

### 2. Prioritize

The user opens the renewal queue and filters or sorts by deadline, risk, owner, or decision status.

Design focus:

- Apply Hick's Law by limiting visible decision choices.
- Use familiar SaaS table and filter patterns.
- Keep related metadata close to each renewal item.

### 3. Investigate

The user opens a subscription detail view to review owner, renewal date, spend, usage signal, notes, approval state, and possible actions.

Design focus:

- Use Miller's Law to group information into manageable sections.
- Use Recognition over Recall by displaying key facts directly.
- Use Law of Common Region to separate summary, usage, spend, and approvals.

### 4. Decide

The user chooses a guided decision path: renew, renegotiate, cancel, consolidate, or request more information.

Design focus:

- Make primary actions easy to reach with Fitts's Law.
- Use Error Prevention for expensive or irreversible choices.
- Keep pending or incomplete decision paths visible with Zeigarnik Effect.

### 5. Confirm

The user reviews a decision summary before finalizing the action.

Design focus:

- Use confirmation patterns for high-impact choices.
- Make the final state clear and reassuring.
- Apply Peak-End Rule by ending the workflow with confidence.

### 6. Track

The user returns to the queue or dashboard and sees updated pending approvals and completed decisions.

Design focus:

- Keep unresolved work visible.
- Show progress without overwhelming the user.
- Reinforce trust through clear status language.

## Journey Risks

- Too many dashboard metrics could reduce clarity.
- Ambiguous owner or approval status could slow action.
- Weak confirmation patterns could lead to accidental decisions.
- Risk and savings indicators could lose meaning if overused.
